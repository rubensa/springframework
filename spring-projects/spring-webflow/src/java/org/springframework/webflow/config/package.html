Contains interfaces and classes for building and assembling web flows.

You construct a web flow using a {@link org.springframework.webflow.config.FlowBuilder}. This package defines the following flow builder implementations:

  • {@link org.springframework.webflow.config.AbstractFlowBuilder} - A convenience superclass to use when you want to assemble the web flow in Java code.
  • {@link org.springframework.webflow.config.XmlFlowBuilder} - A flow builder that reads an XML file containing a web flow definition and constructs the flow accordingly.
During flow construction, a flow builder will need to locate a number of artifacts required by the flow definition: actions, other flows, attribute mappers and so on. To do this, the flow builder uses a configured FlowArtifactFactory. Most Spring applications will use the {@link org.springframework.webflow.config.DefaultFlowArtifactFactory}, which delegates to a standard Spring BeanFactory.

To direct flow construction, you use the {@link org.springframework.webflow.config.registry.FlowAssembler}. This is based on the class GOF Builder design pattern.

Assembled Flows are typically registered in a {@link org.springframework.webflow.config.registry.FlowRegistry} where they may be accessed for execution at runtime. A flow registry is typically populated by using a {@link org.springframework.webflow.config.registry.FlowRegistrar} in conjunction with a {@link org.springframework.webflow.config.registry.AbstractFlowRegistryFactoryBean}.
