Contains an integration layer with Java Server Faces (JSF)

The following capabilities are enabled by this integration:

  • Spring's DelegatingVariableResolver is automatically enabled, which will utilize Spring's standard facilities to create a new bean if the standard JavaServer Faces managed bean facility does not recognize this variable name.
  • An additional resolver is added that maps the (configurable) variable name webApplicationContext to the instance of org.springframework.web.context.WebApplicationContext that Spring has established for this web application.
A sample configuration for faces-config.xml follows:

<faces-config>

  <application>
    <!-- Spring Framework Integration -->
    <variable-resolver>
      org.springframework.web.jsf.DelegatingVariableResolver
    </variable-resolver>
    <!-- Extended "webApplicationContext" resolver -->
    <variable-resolver>
      org.springframework.webflow.jsf.WebApplicationContextVariableResolver
    </variable-resolver>
  </application>

</faces-config>
